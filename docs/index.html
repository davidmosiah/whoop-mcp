Plug-and-play setup

Built for people who do not want to hand-edit JSON.

The setup flow stores credentials locally, creates MCP client snippets, checks readiness, and handles OAuth through a temporary local callback server. 

npx -y whoop-mcp-unofficial setup
npx -y whoop-mcp-unofficial auth
npx -y whoop-mcp-unofficial doctor
01

Create a WHOOP app

Use the WHOOP Developer Dashboard and set the redirect URI to http://127.0.0.1:3000/callback.

02

Run setup

Choose Claude, Cursor, Windsurf, Hermes, OpenClaw, or generic MCP config. Secrets stay in ~/.whoop-mcp/.

03

Ask your agent

Use high-level tools like whoop_daily_summary or deeper collection tools for recovery, sleep and workouts.

Privacy model

Designed for sensitive physiological data.

0600

Local token and config file permissions by default.

local

OAuth tokens stay on your machine and are not returned to the agent.

raw opt-in

Structured mode is default; full raw WHOOP payloads require explicit opt-in.

audit

whoop_privacy_audit reports posture without revealing secrets.

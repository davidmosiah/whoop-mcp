For people who wear WHOOP and use AI assistants

Let your AI understand your recovery, sleep and training.

WHOOP MCP is an unofficial bridge between your WHOOP account and tools like Claude, Cursor, Windsurf, Hermes and OpenClaw. It helps your assistant answer practical questions using your own wearable data, without sending OAuth tokens into chat.

Plain English

What this actually does

1

You connect WHOOP once

You create a small WHOOP developer app, authorize your own account, and the bridge stores the access token locally.

2

Your AI gets useful context

Your assistant can read summaries like recovery, sleep, strain and workouts, then turn them into practical questions and plans.

3

You stay in control

The project is open source, read-only by default, and includes privacy checks so you can see what is stored and where.

Guided setup

Install it in three calm steps

If you can copy and paste commands, you can install this. The details are still here, but the path is intentionally simple.

Step 1

Create a WHOOP developer app

Open the WHOOP Developer Dashboard and create a new app. Use these exact values:

Redirect URL http://127.0.0.1:3000/callback
Scopes read:recovery read:cycles read:workout read:sleep read:profile read:body_measurement

After saving, WHOOP gives you a Client ID and Client Secret. Keep them ready for the next step.

Open developer.whoop.com
Step 2

Run the installer

The CLI asks for your WHOOP credentials, opens the browser for authorization, then checks that everything works.

npx -y whoop-mcp-unofficial setup
npx -y whoop-mcp-unofficial auth
npx -y whoop-mcp-unofficial doctor

When the browser opens, click Authorize in WHOOP. The bridge listens locally on your own machine.

Step 3

Add it to your AI app

Most MCP clients use the same configuration shape. Add this server, restart your client, and ask a WHOOP question.

Claude Cursor Windsurf Hermes OpenClaw
Show MCP config 
{
  "mcpServers": {
    "whoop": {
      "command": "npx",
      "args": ["-y", "whoop-mcp-unofficial"]
    }
  }
}

After install

Ask questions that lead to action

These prompts avoid raw data dumps. They ask your assistant to translate WHOOP signals into decisions.

Morning check-in

Use my latest WHOOP recovery, sleep and strain. Tell me how hard I should train today and what one habit matters most tonight.

Training decision

Look at my last 3 days of workouts and recovery. Should I push, maintain, or deload today? Explain the tradeoff.

Weekly review

Review my last 7 days of WHOOP data. Find the biggest bottleneck and give me a simple plan for next week.

Privacy

Clear boundaries for sensitive health data

Read-only

The bridge asks only for WHOOP read scopes. It does not write workouts, notes, or account changes.

Local tokens

OAuth tokens are stored on your computer under ~/.whoop-mcp/ with user-only file permissions.

No medical claims

This can support coaching and reflection. It is not a medical device, diagnosis tool, or treatment plan.

Auditable

The source is public, and the CLI includes a privacy check so users and agents can verify the setup.
